## TL;DR

HSK2/3相当のリスニング確認テストを回すためのパイプラインを作った。文章をJSONで用意する→edge-ttsで音声化する→HTMLプレイヤーで再生する→解答はClaude Codeに対してインタラクティブに答えて、その場で正誤判定と成績記録をしてもらう、という流れ。

肝は、文章が本当にそのHSKレベルの範囲内かを人間の記憶や目視で決めないこと。最初に手で書いた例文は、実際にはHSK2の範囲外の単語だらけだった。

リポジトリ: https://github.com/r-uehara0219/chinese_listening

## モチベーション

HSKのリスニング教材はたくさんあるが、「選択肢から選ぶ」形式のものが多くて、結局選択肢の消去法で解けてしまう。1文だけ聞いて意味を自分の言葉で言い直す、という一番シビアな聞き取り方をしたかった。

採点をどうするかが問題で、正解の選択肢を用意しない以上、意味が合っているかどうかの判定は人間（か、それに準ずるもの）がやるしかない。ちょうどClaude Codeを使っていたので、「音声を再生する」「答えをテキストで受け取る」までを仕組み化して、判定そのものはインタラクティブにClaude Codeにやらせることにした。

## 何ができるか

- `tests/hsk<N>/*.json` に中国語の文と日本語訳を書く（選択肢なし、1文ずつ）
- `python gen.py` で
  - 文がそのHSKレベルの語彙範囲に収まっているか検証
  - ピンインを自動付与
  - edge-ttsで音声化（男女2声を交互に）
  - `player_data.js` を生成
- `player.html` をブラウザで直接開いて音声を聞き、ボタンで答え（漢字・ピンイン・日本語訳）を表示
- 実際の採点はプレイヤーの外、Claude Codeとの対話で行う。「3番、あなたの携帯は新しい？」のように答えると、Claudeが `tests/` のJSONと照合して正誤判定し、`score.py` で記録する

## ファイル構成

```
tests/hsk2/set_001.json         # 問題データ（level: "HSK2"）
tests/hsk3/set_001.json         # 問題データ（level: "HSK3"）
data/vocab/raw/hsk_2012_L{1,2,3}.txt   # 各レベルで新規に増える公式語彙のみ
data/vocab/extra_allowed.txt    # 全レベル共通の文法的例外
hsk_vocab.py                    # 語彙ゲート（後述）
gen.py                          # 検証 → ピンイン → 音声化 → player_data.js
player.html                     # 再生 + 答え表示のみ
score.py                        # 正誤記録・10問ブロック集計・打ち止め判定
audio/hsk<N>/<set名>/*.mp3      # 生成物。重いのでgitignore
results/hsk<N>/<set名>.jsonl    # 採点履歴
```

音声ファイルは全部で数百KB〜MB単位になるので `.gitignore` で弾いた。クローンした人は `pip install edge-tts pypinyin` して `python gen.py` を叩けば手元で再生成できる。

## 技術的なポイント

### HSK範囲を「判断」せず「検証」する

最初にHSK2相当の例文を8つほど手で書いて、目視で「たぶんHSK2の範囲だろう」と判断していた。実際にチェッカーを作って通してみたら、`火车`（電車）、`自行车`（自転車）、`伞`（傘）、`风`（風）、`色`（色）あたりが軒並み範囲外だった。旧HSK 2.0のHSK1+HSK2の公式語彙は300語しかなく、直感よりかなり狭い。

旧HSK 2.0（2012年版）の公式語彙リストをそのままリポジトリに落として、文がその語彙だけで完全に分割できるかをDPで判定することにした。

```python
def _segment(self, run: str) -> list[str] | None:
    n = len(run)
    back = [None] * (n + 1)
    reachable = [False] * (n + 1)
    reachable[0] = True
    for i in range(1, n + 1):
        for length in range(1, min(self.maxlen, i) + 1):
            if reachable[i - length] and run[i - length : i] in self.words:
                reachable[i] = True
                back[i] = length
                break
    if not reachable[n]:
        return None
    ...
```

分割できなければその文はビルド時に弾かれる（`gen.py --check`）。人間の勘は入らない。

ただし公式リストには「文法的にはHSK2の範囲内だが、単語としては個別に載っていない」形態素がある。`我们`は載っているが`他们`の`们`は単体では載っていない、`没有`はあるが`没去`の`没`はない、といった具合。これは語彙を増やす例外ではなく文法上の穴なので、`data/vocab/extra_allowed.txt` に3文字だけ明示的に許可リストとして書いた。ここに語彙（`半`とか`今年`とか）を足すのは禁止で、あくまで基本形が公式リストにある形態素だけ、というルールにしている。

### レベルをまたいで汎用化する

HSK2の次にHSK3もやる予定があったので、`hsk_vocab.py` はレベル番号を受け取って累積語彙を組み立てる形にした。

```python
def load_level(level: int) -> Vocab:
    allowed: set[str] = set()
    for lv in range(1, level + 1):
        path = RAW / f"hsk_2012_L{lv}.txt"
        allowed |= _load(path)
    extra = _load(DATA / "extra_allowed.txt")
    return Vocab(level, allowed, extra)
```

HSK3のテストはHSK1+2+3の累積（598語）で判定される。ディレクトリも `tests/hsk<N>/` で分け、ディレクトリ名とJSON内の `level` フィールドが食い違っていたらビルドを止めるようにした。新しいレベルを足すときは公式語彙ファイルを1個置いてディレクトリを作るだけで済む。

実際にHSK3の例文を書いたときも、`忘`（`忘记`でしか登録されていない）、`饭`（`米饭`はあるが単体の`饭`はない）あたりを手癖で使って弾かれた。何回作ってもこのパターンで引っかかるので、機械検証にした判断は間違っていなかったと思う。

### 採点とマスタリー基準

採点自体はClaude Codeとの対話でやるので特別な実装は要らないが、「どこまでやったら次に進んでいいか」の基準だけは欲しかった。`score.py` に正誤を1問ずつ食わせると、10問ブロック単位で集計して、直近の完全なブロックが90%（9/10か10/10）を超えたら `>> mastery reached` と出すようにした。

```
$ python score.py add hsk2/set_001 8 0
[hsk2/set_001] 3/8 correct overall (38%)
    1-8   3/8  [xxooxxox]  <- latest
```

記録は `results/hsk<N>/<set名>.jsonl` にJSON Lines形式で溜まるので、セッションをまたいでも `results/**/*.jsonl` を読めばそれまでの成績が分かる。

## 既知の問題

- DPによる語彙チェックは「文字単位の分割可能性」しか見ていないので、たとえば `男朋友` のように公式リストに載っていない複合語でも、`男`＋`朋友` のように個別の単語に分解できてしまえば通ってしまう。今のところ実害はないが、意味的に正しい複合語かどうかまでは保証しない
- ピンインは `pypinyin` のデフォルト（多音字の文脈判定なし）に任せているので、まれに読み間違いが起きうる
- 音声は毎回生成し直すとedge-ttsへのリクエストが飛ぶので、既存ファイルがあればスキップする実装にしている。声質を変えたいときは `--force`

## まとめ

「聞いた文の意味を自分の言葉で答える」形式にしたかったのと、「HSKの範囲内かどうかを機械的に検証したい」の2つを満たすために、生成・音声化・プレイヤー・採点を分業させるパイプラインを組んだ。一番効いたのは語彙ゲートで、これがなかったらなんちゃってHSK2の教材を作って気づかず練習することになっていたと思う。次はHSK4以降を同じ仕組みで足していく予定。
