# HSK リスニング確認テスト / 単語想起テスト

二種類の確認テストがある:

1. **リスニング** -- 1文ずつ中国語を聞き、その意味を Claude Code に口頭（テキスト）で答え、Claude がその場で正誤判定する。
2. **単語想起** -- 日本語の例文（対象語を太字）を見て、対応する中国語の単語をピンイン＋声調番号（例: `piaoliang 4`）で答え、Claude が正誤判定する。音声は使わない。

HSK2以降、レベルごとにディレクトリを分けて拡張していく想定（現在は HSK2 のみ実施中）。

## 構成（リスニング）

| パス | 役割 |
| --- | --- |
| `tests/hsk<N>/*.json` | 問題データ。`{id, title, level: "HSK<N>", sentences: [{n, zh, ja}]}` のみ手で書く |
| `data/vocab/raw/hsk_2012_L<N>.txt` | 公式データ。レベルNで**新規**に増える語のみ（旧HSK 2.0, 2012年版） |
| `data/vocab/extra_allowed.txt` | 全レベル共通の明示的な例外（文法的形態素のみ。現状 `们` `没` `儿`） |
| `hsk_vocab.py` | 語彙ゲート。`load_level(N)` が HSK1..N の**累積**語彙を組み立て、DPで文を分割できるか判定 |
| `gen.py` | `tests/hsk<N>/` を全走査 → 語彙検証 → ピンイン生成 → edge-tts音声化 → `player_data.js` 出力 |
| `player.html` | プレイヤー。音声再生と答え表示のみ（採点はしない）。セット選択はレベルごとにグループ表示 |
| `score.py` | 正誤記録・10問ブロック集計・打ち止め判定（リスニング・単語想起で共用） |

音声は `audio/hsk<N>/<set名>/s<NN>.mp3`、成績は `results/hsk<N>/<set名>.jsonl` に、テストと同じ木構造で保存される。

## 構成（単語想起）

| パス | 役割 |
| --- | --- |
| `vocab/hsk<N>/*.json` | 問題データ。`{id, title, level: "HSK<N>", items: [{n, ja, word}]}` のみ手で書く。`ja` は対象語を `**太字**` で囲んだ日本語例文、`word` は対象の中国語単語のみ（ピンインは書かない） |
| `gen_vocab.py` | `vocab/hsk<N>/` を全走査 → 語彙検証 → pypinyinでピンイン・声調を機械生成 → `vocab_data.js` 出力 |

```
python gen_vocab.py            # vocab/hsk<N>/*.json をビルド
python gen_vocab.py --check    # 語彙検証のみ
```

**単語想起の語彙範囲はリスニングと異なり累積ではなく、そのレベルで新規に増える語のみ**（`data/vocab/raw/hsk_2012_L<N>.txt` との**完全一致**）。HSK1の語をHSK2の単語想起テストに混ぜてはいけない（すでに前段階で扱った語を再度出すことになるため）。`gen_vocab.py --check` が単語がその公式リストに完全一致するかを検証する。

声調はpypinyinの`Style.TONE3`（`neutral_tone_with_five=True`）で機械判定し、軽声（5）の音節は圧縮表記から省略する（例: `piaoliang`は両音節とも本来の声調が返るため`4`ではなく`44`、`意思`は2音節目が軽声なので`4`のみ）。**声調を手で判断してはいけない**（軽声かどうかは辞書依存で直感に反することがある）。

## 採点フロー（単語想起・Claude の仕事）

1. Claude が `vocab_data.js`（または `vocab/hsk<N>/*.json` + 機械計算）から1問ずつ `ja` を提示する
2. ユーザーが中国語の単語とピンイン＋声調番号で答える（例:「漂亮 piaoliang 44」）
3. Claude が `word`・`pinyin`・`tones_compact` と照合し正誤判定する（声調記号の有無・大文字小文字は無視）
4. `python score.py add vocab/hsk<N>/<set名> <n> <0|1>` で記録する（リスニングと同じ `score.py` を共用、`set_id` の先頭に `vocab/` を付けて名前空間を分ける）

## ビルド

```
python gen.py            # tests/hsk<N>/*.json を全部ビルド（既存音声はスキップ）
python gen.py --check    # 語彙検証のみ、音声もplayer_data.jsも生成しない
python gen.py --force    # 音声を再生成
python gen.py --rate -20%  # 読み上げを遅く
```

`player.html` をブラウザで直接開く（サーバー不要。`player_data.js` は `file://` でも読める `.js` 代入形式）。

## 語彙の範囲は「判断」せず「検証」する

**HSKの範囲内かどうかを記憶や直感で決めてはいけない。** 過去にそれをやって（HSK2のつもりで）`火车` `自行车` `色` `伞` `风` `冰` `骑` `带` などを混入させたことがある。

必ず `python gen.py --check` を通すこと。範囲外の語があればビルドが失敗する。

`hsk_vocab.load_level(N)` は **HSK1〜Nの累積**を返す（例: HSK3なら HSK1+2+3の全語彙、2026-07-15時点で301語→598語）。上位レベルは下位レベルの語を含む厳密な上位集合になる。

HSK2の語彙リストの実態として、直感に反する欠落がある(参考。他レベルにも同様の落とし穴があり得る):

- `天` が無い → **`每天` は使えない**（`今天`/`明天`/`昨天` はある）
- `火车` が無い → `火车站` のみ可
- `半` `今年` `杯` `色` `外面` `然后` `可是` が無い
- `跑` `唱` 単独が無い → `跑步`/`唱歌` はあるが `跑得很快` `唱得很好` は不可

`data/vocab/extra_allowed.txt` に足してよいのは、基本形がどこかの公式リストにある**文法的形態素だけ**（`我们`→`们`、`没有`→`没`、`一点儿`→`儿`）。語彙そのものを増やすもの（`天` `半` など）を足して範囲を広げてはいけない。この例外リストは全レベル共通・累積で効く。

## 新しいHSKレベルを追加する手順

1. `data/vocab/raw/hsk_2012_L<N>.txt` に、そのレベルで新規に増える公式語彙リストを置く（`https://raw.githubusercontent.com/glxxyz/hskhsk.com/main/data/lists/HSK%20Official%202012%20L<N>.txt` などから取得。既存のL1〜L3は取得済み）
2. `tests/hsk<N>/` ディレクトリを作り、`{"level": "HSK<N>", ...}` を持つJSONを置く
3. `python gen.py --check` で範囲外の語がないか確認してから `python gen.py` を実行

ディレクトリ名の `hsk<N>` と JSON内の `"level": "HSK<N>"` は一致していないとビルドが失敗する（食い違いを検知するための意図的なチェック）。

## 採点フロー（リスニング・Claude の仕事）

1. ユーザーが `player.html` で第N文を聞く
2. ユーザーが意味を日本語で答える(例:「3番、あなたの携帯は新しい？」)
3. Claude が該当セットの `tests/hsk<N>/*.json` 内、該当 `n` の `ja` / `zh` と照合して正誤を判定する
4. 判定結果を `python score.py add <set_id> <n> <0|1>` で記録する(正解=1、不正解=0)。
   `<set_id>` は `tests/` からの相対パス(拡張子抜き)。例: `hsk2/set_001`。
   実行すると累計と直近ブロックの集計が表示されるので、その出力をそのままユーザーに伝える。

判定は文字列一致ではなく**意味が取れているか**で見る。訳語の揺れ(「携帯」「スマホ」等)は正解。
主語・時制・否定・数量・疑問かどうかを取り違えていたら誤りとして指摘し、`zh` とピンインを示して聞き取れなかった箇所を説明する。

## 成績記録

`results/<set_id>.jsonl` に採点順で1行ずつ記録される(`{"n": ..., "correct": ...}`)。`<set_id>` にディレクトリ階層を含められるため、リスニングは `hsk2/set_001`、単語想起は `vocab/hsk2/set_001` のように名前空間を分ければ同名セットでも記録は混ざらない。
`score.py` は常に10問ブロック単位で集計を表示する。

```
python score.py add <set_id> <n> <0|1>   # 1問記録して集計を表示
python score.py summary <set_id>          # 記録済みの集計だけ再表示
python score.py reset <set_id>            # そのセットの記録を消す(要ユーザー確認)
```

新しいセッションでも `results/**/*.jsonl` を読めばこれまでの成績が分かる。`reset` は破壊的操作なので、ユーザーが明示的に望んだときだけ実行する。

## 打ち止め基準

直近の完全な10問ブロックが **90%以上(9/10 or 10/10)** なら「打ち止めていい」の合図。
`score.py add` / `summary` はブロックが完成するたびに `>> mastery reached` か `>> keep going` を自動表示するので、
それが出たらユーザーにそのまま伝える。まだ10問に満たない途中のブロックでは判定しない。
